MERN Blog API - Documentation

Base URL: /api/v1

Authentication

POST /auth/register

Registers a new user and sends a verification email.

Request Body: 
{
    "username": "testuser",
    "email": "test@example.com",
    "password": "password123"
}
POST /auth/login

Logs in a verified user. Returns a short-lived access token in the response body and a long-lived refresh token in a secure, httpOnly cookie.

Request Body: 
{
    "email": "test@example.com",
    "password": "password123"
}
GET /auth/refresh

Uses the httpOnly refresh token cookie to generate a new access token.

POST /auth/logout

Logs out the user by clearing the refresh token from the database and the client's cookie.

GET /auth/verify-email?token={token}

Verifies a user's email using the token sent to their inbox.

POST /auth/resend-verification

Resends the verification email to an unverified user.

Request Body: 
{ "email": "unverified@example.com" }
POST /auth/forgot-password

Sends a password reset link to the user's email.

Request Body: 
{ "email": "user@example.com" }
POST /auth/reset-password?token={token}

Resets the user's password using the reset token.

Request Body: 
{ "password": "newStrongPassword123" }
GET /auth/me Auth Required

Gets the profile of the currently logged-in user.

PUT /auth/me Auth Required

Updates the profile of the currently logged-in user.

Request Body: 
{
    "bio": "A short bio about me.",
    "profilePicture": "https://example.com/image.jpg",
    "socialLinks": { "twitter": "https://twitter.com/user" }
}

Posts

POST /posts Auth Required

Creates a new blog post.

Request Body: 
{
    "title": "My New Post",
    "content": "This is the content of the post.",
    "excerpt": "Short summary of the post.",
    "coverImage": "https://example.com/image.jpg",
    "category": "60d0fe4f5311236168a109ca",
    "tags": ["tag1", "tag2"],
    "status": "published",
    "isFeatured": false
}
GET /posts

Gets a paginated list of all published posts. Supports filtering by ?tag= and ?category=.

GET /posts/featured

Gets a list of featured posts.

GET /posts/popular

Gets a list of popular posts (by views/likes).

GET /posts/:slug

Gets a single post by its slug and increments its view count.

PUT /posts/:postId Auth Required

Updates a post. User must be the author or an admin.

Request Body: 
{
    "title": "Updated Post Title",
    "status": "published",
    "isFeatured": true
}
DELETE /posts/:postId Auth Required

Deletes a post. User must be the author or an admin.

POST /posts/:postId/like Auth Required

Toggles a like on a post for the logged-in user.

GET /posts/:postId/related

Gets related posts based on tags.

Comments

POST /comments Auth Required

Creates a new comment on a post. Can be a top-level comment or a reply.

Request Body: 
{
    "postId": "60d0fe4f5311236168a109ca",
    "content": "This is a great post!",
    "parentComment": null 
    // or "parentComment": "60d0fe4f5311236168a109cb" to reply
}
GET /comments/post/:postId

Gets all comments for a specific post in a nested structure.

PUT /comments/:commentId Auth Required

Updates a comment. User must be the author.

DELETE /comments/:commentId Auth Required

Deletes a comment. User must be the author or an admin.

Categories

POST /categories Admin Only

Creates a new category.

Request Body: 
{ "name": "Technology" }
GET /categories

Gets a list of all categories.

GET /categories/:slug/posts

Gets all published posts belonging to a specific category slug.

Users

GET /users/author/:authorId

Gets the public profile of an author.

GET /users/author/:authorId/posts

Gets all published posts by a specific author.

Misc

GET /search?q={query}

Searches for posts by title and content.

POST /subscribe

Subscribes an email to the newsletter.

Request Body: 
{ "email": "subscriber@example.com" }
GET /tags

Gets a list of all unique tags used in published posts.
